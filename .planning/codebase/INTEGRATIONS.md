# External Integrations

**Analysis Date:** 2026-04-18

## APIs & External Services

**Browser Platform:**
- Chrome Extension runtime messaging and lifecycle - `background.js` handles `runtime.onMessage`, `runtime.onInstalled`, and `runtime.onStartup`, while `popup.js` and `dashboard.js` call `runtime.sendMessage`
  - SDK/Client: browser-provided `globalThis.browser ?? chrome` in `background.js`, `dashboard.js`, and `popup.js`
  - Auth: Not applicable
- Chrome Tabs and Windows APIs - `dashboard.js` captures tabs with `tabs.query`, and `background.js` reopens saved sessions with `windows.create` and `tabs.create`
  - SDK/Client: browser-provided `chrome.tabs` and `chrome.windows`
  - Auth: extension permission declared in `manifest.json` (`tabs`)
- Chrome Bookmarks API - `background.js` creates the `"Browsing Library"` folder tree, imports existing bookmark folders, moves bookmarks after AI recategorization, and deletes bookmark nodes
  - SDK/Client: browser-provided `chrome.bookmarks`
  - Auth: extension permission declared in `manifest.json` (`bookmarks`)
- Chrome Storage API - `background.js` and `dashboard.js` persist local state and cross-device sync state with `chrome.storage.local`, `chrome.storage.sync`, and `storage.onChanged`
  - SDK/Client: browser-provided `chrome.storage`
  - Auth: extension permission declared in `manifest.json` (`storage`)

**AI Enrichment:**
- Google Generative Language API (Gemini) - optional category, tag, description, and summary generation is requested directly from `dashboard.js` and `background.js`
  - SDK/Client: raw `fetch` in `dashboard.js` and `background.js`; no third-party SDK is used
  - Auth: user-supplied API key stored in `chrome.storage.local` under `tabLedgerSettings` by `dashboard.js`, sent as `x-goog-api-key`, and allowed by the host permission in `manifest.json`
- Google AI Studio setup link - the settings panel in `dashboard.html` links users to `https://aistudio.google.com/app/apikey`
  - SDK/Client: plain anchor in `dashboard.html`
  - Auth: handled outside the extension

**Local Browser Data Sources:**
- Open tabs across all normal browser windows - read in `dashboard.js`
  - SDK/Client: `chrome.tabs`
  - Auth: extension permission declared in `manifest.json` (`tabs`)
- Existing bookmark folders and bookmark items - read and transformed into sessions in `background.js`
  - SDK/Client: `chrome.bookmarks.getTree()` and `chrome.bookmarks.getSubTree()` in `background.js`
  - Auth: extension permission declared in `manifest.json` (`bookmarks`)

## Data Storage

**Databases:**
- None
  - Connection: Not applicable
  - Client: Not applicable

**Browser Storage:**
- `chrome.storage.local` is the authoritative app store in `background.js` and `dashboard.js`
  - Connection: browser-managed local extension storage
  - Client: `chrome.storage.local`
- Stored keys in `chrome.storage.local`:
  - `tabGardenDraft` - draft capture state from `dashboard.js`
  - `bookmarkMetadata` - bookmark item metadata maintained by `background.js`
  - `savedSessions` - saved library sessions maintained by `background.js` and read by `dashboard.js`
  - `aiEnrichmentQueue` and `aiEnrichmentControl` - background AI queue state in `background.js`
  - `tabLedgerSettings` - dedupe settings, Gemini API key, and Gemini model written by `dashboard.js`
  - `deviceId` - locally generated sync identity written by `background.js`
- `chrome.storage.sync` stores lightweight cross-device session summaries in `background.js`
  - Connection: browser-managed sync extension storage
  - Client: `chrome.storage.sync`
- Stored keys in `chrome.storage.sync`:
  - `sync:session:*` - per-session summary payloads written by `background.js`
  - `sync:index` - current local session ID list written by `background.js`

**File Storage:**
- Chrome bookmarks tree is used as the primary persistent library store in `background.js`
- Local filesystem export only - `dashboard.js` creates downloadable JSON files with `Blob` and `URL.createObjectURL`
- Packaging artifacts are generated by `package-extension.sh` under `output/package/` and copied to `/mnt/c/Users/omri/Documents/TabLedger-Chrome-Ext`

**Caching:**
- No dedicated cache layer is detected beyond `chrome.storage.local` and `chrome.storage.sync`

## Authentication & Identity

**Auth Provider:**
- None detected
  - Implementation: the extension does not sign users in; it only accepts an optional Gemini API key through the settings UI in `dashboard.html` and `dashboard.js`

**Identity:**
- Local device identity only
  - Implementation: `background.js` generates a `deviceId` and stores it in `chrome.storage.local` so synced sessions can track which device created them
